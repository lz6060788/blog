## 新增需求

### 需求：数据库连接初始化
系统应在应用启动时使用 Drizzle ORM 初始化 SQLite 数据库连接。

#### 场景：成功初始化数据库
- **当** 应用启动时
- **则** 如果不存在，则在 `./data/db.sqlite` 处创建 SQLite 数据库文件
- **且** 单例数据库客户端实例可供导入使用
- **且** 连接配置为使用 Node.js 运行时

### 需求：ORM 客户端可访问性
系统应提供可从 lib 模块导出的集中式数据库客户端。

#### 场景：导入数据库客户端
- **当** 任何模块从 `@/lib/db` 导入时
- **则** Drizzle 数据库客户端可用
- **且** 客户端具有数据库 schema 的类型

#### 场景：复用现有连接
- **当** 同一请求中多次导入 `@/lib/db` 时
- **则** 复用同一个数据库连接实例

### 需求：Schema 定义能力
系统应支持使用 Drizzle ORM schema 构建器定义数据库 schema。

#### 场景：定义表 schema
- **当** 开发者使用 Drizzle schema 构建器创建 schema 文件时
- **则** 可以定义具有列、关系和约束的表
- **且** TypeScript 类型自动推断

### 需求：迁移系统
系统应通过 Drizzle Kit 支持数据库 schema 迁移。

#### 场景：从 schema 生成迁移
- **当** 开发者运行 `drizzle-kit generate` 时
- **则** 基于 schema 变更生成迁移 SQL 文件
- **且** 迁移文件存储在迁移目录中

#### 场景：将迁移应用到数据库
- **当** 开发者运行 `drizzle-kit migrate` 时
- **则** 将待处理的迁移应用到数据库
- **且** 跟踪迁移历史

#### 场景：在开发中推送 schema 变更
- **当** 开发者运行 `drizzle-kit push` 时
- **则** 直接更新数据库 schema 而无需迁移文件
- **且** 这用于开发中的快速原型设计

### 需求：数据库驱动配置
系统应使用 `better-sqlite3` 作为 SQLite 驱动。

#### 场景：安装驱动
- **当** 安装依赖时
- **则** `better-sqlite3` 包可用
- **且** 安装 `@types/better-sqlite3` 以支持 TypeScript

#### 场景：配置驱动
- **当** 初始化数据库连接时
- **则** 使用 `better-sqlite3` 作为底层数据库驱动
