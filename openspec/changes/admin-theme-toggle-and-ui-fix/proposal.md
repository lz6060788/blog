## Why

管理端当前缺少主题切换功能，且无法与主站的主题状态同步。无论用户在主站选择何种主题，进入管理端后总是显示浅色主题，造成体验不一致。此外，存在多处 UI 规范问题：浅色主题下按钮悬浮时文字颜色与背景色对比度不足，新建文章、发布等按钮缺少悬浮状态反馈，违背了 UI 设计的基本可访问性和交互反馈原则。

## What Changes

- **管理端主题切换功能**：在管理端顶部栏添加主题切换组件，允许用户在浅色、深色和自动模式之间切换
- **主题状态同步**：确保管理端与主站共享同一 localStorage 主题配置，实现主题状态在两个区域间的同步
- **UI 规范修复**：修复浅色主题下按钮悬浮状态的对比度问题，确保文字颜色与背景色符合 WCAG AA 标准（至少 4.5:1）
- **交互反馈完善**：为所有交互元素添加适当的悬浮、焦点和激活状态样式
- **组件 UI 规范检查与修复**：审查项目中所有组件的 CSS 类名和样式，确保使用主题变量而非硬编码颜色，并为所有用户交互行为添加视觉反馈

## Capabilities

### New Capabilities

- **admin-theme-sync**: 管理端主题切换与同步功能，包括主题切换 UI 组件、localStorage 同步机制、以及管理端与主站之间的主题状态共享

### Modified Capabilities

- **theme-system**: 扩展现有主题系统规范，确保管理端能够正确使用主题变量，并修复 UI 规范相关问题（对比度、交互反馈等）

## Impact

- **管理端布局** (`app/admin/layout.tsx`)：添加 ThemeProvider 包装
- **管理端顶部栏** (`components/admin/top-bar.tsx`)：集成 ThemeToggle 组件
- **按钮组件** (`components/ui/button.tsx`)：修复悬浮状态的颜色对比度问题
- **所有管理端页面组件**：审查并修复不符合 UI 规范的样式
- **主题系统规范** (`openspec/specs/theme-system/spec.md`)：通过 delta spec 更新相关要求
