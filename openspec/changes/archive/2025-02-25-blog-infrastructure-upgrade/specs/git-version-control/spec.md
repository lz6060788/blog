## 新增需求

### 需求：Git 仓库初始化
系统应在项目根目录中初始化 Git 仓库，并进行适当的配置。

#### 场景：初始 Git 设置
- **当** 执行初始化命令时
- **那么** 系统应在项目根目录创建 .git 目录
- **并且** 系统应准备好进行版本控制操作

### 需求：GitHub 远程连接
系统应将本地 Git 仓库连接到远程 GitHub 仓库 `https://github.com/lz6060788/blog`。

#### 场景：远程仓库连接
- **当** 配置远程源时
- **那么** 系统应将远程 URL 设置为 `https://github.com/lz6060788/blog`
- **并且** 系统应能够从远程仓库推送和拉取

### 需求：Gitignore 配置
系统应创建一个全面的 .gitignore 文件，排除常见的 Next.js 构建产物、依赖项和环境文件。

#### 场景：排除构建产物
- **当** 创建 .gitignore 文件时
- **那么** 系统应排除 .next/、node_modules/、dist/ 和 build 目录
- **并且** 系统应排除 .env.local、.env.development.local 和 .env.test.local 文件
- **并且** 系统应排除系统文件，如 .DS_Store 和 Thumbs.db

### 需求：初始提交
系统应创建一个包含所有项目源文件的初始提交。

#### 场景：首次提交创建
- **当** 创建初始提交时
- **那么** 系统应暂存所有项目文件，遵循 .gitignore 规则
- **并且** 系统应创建一个描述初始项目设置的提交消息
- **并且** 系统应将提交分配给 main 分支

### 需求：分支管理准备
系统应使用 main 分支作为主要开发分支来配置仓库。

#### 场景：主分支设置
- **当** 初始化仓库时
- **那么** 系统应使用 "main" 作为默认分支名称
- **并且** 系统应准备好功能分支工作流
