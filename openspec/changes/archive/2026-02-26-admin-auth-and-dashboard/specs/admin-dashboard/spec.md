## ADDED Requirements

### 需求：后台首页概览

系统应在 `/admin` 路径提供后台首页，显示博客的基本统计和快捷操作。

#### 场景：访问后台首页

- **当** 用户访问 `/admin`
- **则** 系统显示后台首页概览
- **且** 页面显示以下统计信息：
  - 文章总数（已发布 + 草稿）
  - 已发布文章数
  - 草稿数
  - 最近 7 天新增文章数

#### 场景：快捷操作卡片

- **当** 后台首页被渲染
- **则** 页面显示快捷操作卡片：
  - "新建文章"按钮（链接到 `/admin/posts/new`）
  - "查看草稿"按钮（链接到 `/admin/drafts`）
  - "管理文章"按钮（链接到 `/admin/posts`）

#### 场景：最近活动列表

- **当** 后台首页被渲染
- **则** 页面显示最近活动列表
- **且** 列表显示最近 5 条操作记录（如：创建了某文章、发布了某文章）
- **且** 每条记录显示操作时间和操作内容

### 需求：导航菜单高亮

系统应根据当前页面高亮对应的侧边栏导航项。

#### 场景：后台首页激活状态

- **当** 用户访问 `/admin`
- **则** 侧边栏的"首页"或"概览"导航项显示高亮状态
- **且** 高亮样式与其他导航项明显区分

#### 场景：文章管理页激活状态

- **当** 用户访问 `/admin/posts` 或其子页面（如 `/admin/posts/[id]/edit`）
- **则** 侧边栏的"文章管理"导航项显示高亮状态

#### 场景：草稿箱页激活状态

- **当** 用户访问 `/admin/drafts`
- **则** 侧边栏的"草稿箱"导航项显示高亮状态

### 需求：设置页面（基础版）

系统应在 `/admin/settings` 路径提供基础设置页面。

#### 场景：访问设置页面

- **当** 用户访问 `/admin/settings`
- **则** 显示设置表单
- **且** 表单包含以下配置项：
  - 博客名称
  - 博客描述
  - 每页显示文章数
  - 显示/隐藏登录按钮（如果需要为前台保留此功能）

#### 场景：保存设置

- **当** 用户修改设置并点击"保存"
- **则** 系统更新配置文件或数据库
- **且** 显示"设置已保存"成功提示
- **且** 设置立即生效

### 需求：面包屑导航

系统应在后台页面显示面包屑导航，帮助用户了解当前位置。

#### 场景：显示面包屑

- **当** 用户在后台页面浏览
- **则** 顶部栏下方显示面包屑导航
- **且** 面包屑格式为"首页 / 文章管理 / 编辑文章"
- **且** 面包屑的每一项都是可点击的链接

#### 场景：首页面包屑

- **当** 用户在后台首页
- **则** 面包屑只显示"首页"
- **且** 不显示链接（因为已在首页）
