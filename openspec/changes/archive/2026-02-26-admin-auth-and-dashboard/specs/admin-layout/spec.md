## ADDED Requirements

### 需求：后台布局结构

系统应为后台管理系统提供独立的布局组件，包含侧边栏、顶部栏和内容区域。

#### 场景：后台布局渲染

- **当** 用户访问任何 `/admin/*` 路径
- **则** 系统渲染 `app/admin/layout.tsx` 布局组件
- **且** 布局包含左侧固定侧边栏（宽度 250px）
- **且** 布局包含顶部固定导航栏（高度 60px）
- **且** 布局包含右侧可滚动内容区域

#### 场景：侧边栏导航菜单

- **当** 后台布局被渲染
- **则** 侧边栏显示以下导航项：
  - 文章管理（链接到 `/admin/posts`）
  - 草稿箱（链接到 `/admin/drafts`）
  - 设置（链接到 `/admin/settings`）
- **且** 当前活动页面对应的导航项显示高亮状态
- **且** 导航项支持键盘 Tab 键导航

#### 场景：顶部栏用户信息

- **当** 后台布局被渲染
- **则** 顶部栏右侧显示当前用户的头像和名称
- **且** 用户头像使用 OAuth 提供商返回的图片 URL
- **且** 用户名称显示为 `name` 或 `email` 字段（优先 name）
- **且** 点击用户信息显示下拉菜单，包含"登出"选项

### 需求：响应式布局适配

系统应确保后台布局在不同屏幕尺寸下正常显示。

#### 场景：桌面端显示

- **当** 用户在桌面端（宽度 >= 1024px）访问后台
- **则** 侧边栏固定显示在左侧
- **且** 内容区域占据剩余空间
- **且** 侧边栏和内容区域并排显示

#### 场景：移动端侧边栏折叠

- **当** 用户在移动端（宽度 < 1024px）访问后台
- **则** 侧边栏默认隐藏
- **且** 顶部栏左侧显示汉堡菜单按钮
- **且** 点击汉堡菜单按钮展开侧边栏（从左侧滑入）
- **且** 点击内容区域或按 ESC 键关闭侧边栏

### 需求：登出功能

系统应允许用户从后台系统登出。

#### 场景：点击登出按钮

- **当** 用户点击顶部栏的"登出"按钮
- **则** 系统调用 NextAuth.js 的 `signOut` 函数
- **且** 清除用户会话
- **且** 将用户重定向到博客首页（`/`）

#### 场景：登出后访问后台

- **当** 用户登出后尝试访问 `/admin`
- **则** 系统将用户重定向到 `/login`
- **且** 用户需要重新登录才能访问后台
