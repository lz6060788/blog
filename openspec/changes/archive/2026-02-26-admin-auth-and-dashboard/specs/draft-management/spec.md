## ADDED Requirements

### 需求：草稿箱页面

系统应在 `/admin/drafts` 路径提供草稿管理页面，专门显示未发布的文章。

#### 场景：访问草稿箱

- **当** 用户访问 `/admin/drafts`
- **则** 系统从数据库查询所有草稿（`published: false`）
- **且** 使用表格组件显示草稿列表
- **且** 每行显示：标题、最后修改时间、操作按钮

#### 场景：草稿列表操作

- **当** 草稿列表被渲染
- **则** 每篇草稿行包含以下操作：
  - "编辑"按钮（链接到 `/admin/posts/[id]/edit`）
  - "发布"按钮（直接发布文章）
  - "删除"按钮（触发确认对话框）

#### 场景：从草稿箱发布文章

- **当** 用户点击草稿的"发布"按钮
- **则** 系统更新数据库中该文章的 `published` 字段为 `true`
- **且** 文章从草稿箱移除
- **且** 显示"文章已发布"成功提示
- **且** 自动刷新草稿列表

### 需求：自动保存草稿

系统应在用户编辑文章时自动保存草稿，防止内容丢失。

#### 场景：自动保存触发

- **当** 用户在编辑页面修改文章内容
- **则** 系统每 30 秒自动保存一次草稿
- **且** 自动保存时显示"正在保存..."提示
- **且** 保存成功后显示"已自动保存"提示（2 秒后消失）

#### 场景：自动保存失败处理

- **当** 自动保存请求失败（网络错误或服务器错误）
- **则** 显示"自动保存失败，请检查网络连接"错误提示
- **且** 保留编辑内容在本地
- **且** 用户仍可手动保存

### 需求：草稿与已发布文章区分

系统应在后台明确区分草稿和已发布文章。

#### 场景：文章列表显示状态标识

- **当** 用户在文章列表页面查看文章
- **则** 已发布文章显示绿色"已发布"标签
- **且** 草稿文章显示灰色"草稿"标签
- **且** 标签位于文章标题旁边

#### 场景：草稿文章在列表中的排序

- **当** 用户查看文章列表
- **则** 草稿文章默认排在已发布文章之后
- **且** 同类型文章按最后修改时间倒序排列

### 需求：空草稿箱提示

系统应在草稿箱为空时显示友好的提示。

#### 场景：无草稿时显示提示

- **当** 数据库中没有任何草稿
- **且** 用户访问草稿箱页面
- **则** 显示空状态提示
- **且** 提示文案为"还没有任何草稿，开始写你的第一篇文章吧"
- **且** 显示"新建文章"按钮

### 需求：草稿预览

系统应允许用户预览草稿文章在前台的实际显示效果。

#### 场景：预览草稿

- **当** 用户在编辑页面点击"预览"按钮
- **则** 系统在新标签页打开文章预览页面
- **且** 预览页面显示文章的实际渲染效果
- **且** 预览页面顶部显示"预览模式"提示栏
- **且** 未登录用户无法通过 URL 直接访问预览页面
