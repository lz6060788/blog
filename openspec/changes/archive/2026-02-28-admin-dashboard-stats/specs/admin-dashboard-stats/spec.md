## ADDED Requirements

### Requirement: 获取文章总数统计
系统应提供统计所有文章总数的能力，包括已发布和草稿文章。

#### 场景：获取文章总数
- **当** 管理端请求文章统计数据时
- **则** 系统返回数据库中所有文章的总数
- **且** 统计包括已发布和未发布的文章

#### 场景：统计结果为整数
- **当** 文章统计查询执行时
- **则** 系统返回一个整数值
- **且** 该值代表数据库中 posts 表的记录总数

### Requirement: 获取已发布文章统计
系统应提供统计已发布文章数量的能力。

#### 场景：获取已发布文章数
- **当** 管理端请求已发布文章统计时
- **则** 系统返回 published=true 的文章数量
- **且** 仅统计已发布的文章

### Requirement: 获取草稿文章统计
系统应提供统计草稿文章数量的能力。

#### 场景：获取草稿文章数
- **当** 管理端请求草稿文章统计时
- **则** 系统返回 published=false 的文章数量
- **且** 仅统计未发布的文章

### Requirement: 获取最近7天新增文章统计
系统应提供统计最近7天内新增文章数量的能力。

#### 场景：获取最近7天新增文章数
- **当** 管理端请求最近7天统计时
- **则** 系统返回最近7天内创建的文章数量
- **且** 时间计算基于当前时间向前推算7天
- **且** 判断依据是文章的 createdAt 字段

#### 场景：最近7天时间计算
- **当** 计算最近7天时
- **则** 系统使用 7天 × 24小时 × 60分钟 × 60秒 × 1000毫秒 = 604800000 毫秒
- **且** 起始时间为当前时间减去604800000毫秒
- **且** 统计 createdAt 大于等于起始时间的文章

### Requirement: 统计数据聚合接口
系统应提供统一的接口返回所有管理端首页所需的统计数据。

#### 场景：获取完整统计数据
- **当** 管理端首页加载时
- **则** 系统返回包含所有统计指标的数据结构
- **且** 数据结构包含文章总数、已发布数、草稿数、最近7天新增数
- **且** 响应为 JSON 格式

#### 场景：统计数据结构
- **当** 返回统计数据时
- **则** 响应包含以下字段：
  - `totalPosts`: 文章总数（整数）
  - `publishedPosts`: 已发布文章数（整数）
  - `draftPosts`: 草稿文章数（整数）
  - `recentPosts`: 最近7天新增文章数（整数）

### Requirement: 统计查询性能
系统应确保统计查询在合理时间内完成。

#### 场景：统计查询执行时间
- **当** 执行统计查询时
- **则** 查询应在100毫秒内完成
- **且** 使用数据库的 COUNT 聚合函数而非全表扫描

### Requirement: 统计数据一致性
系统应确保统计数据与实际数据库状态一致。

#### 场景：数据一致性验证
- **当** 同时获取多个统计指标时
- **则** totalPosts 应等于 publishedPosts 加 draftPosts
- **且** 所有统计基于同一时刻的数据库快照
