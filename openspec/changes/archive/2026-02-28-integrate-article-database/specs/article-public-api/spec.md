## 新增需求

### 需求：获取已发布文章列表
系统应提供从数据库获取已发布文章列表的能力，仅返回 published 状态为 true 的文章。

#### 场景：获取所有已发布文章
- **当** 前台页面请求文章列表时
- **那么** 系统应从数据库查询所有 published=true 的文章
- **并且** 系统应按发布日期倒序排列
- **并且** 每篇文章应包含 id、title、excerpt、date、category、tags、readTime

#### 场景：按分类筛选文章
- **当** 用户指定分类筛选时
- **那么** 系统应返回该分类下已发布的文章
- **并且** 未指定分类时返回所有已发布文章

#### 场景：按标签筛选文章
- **当** 用户指定标签筛选时
- **那么** 系统应返回包含该标签的已发布文章

### 需求：获取单篇文章详情
系统应根据文章 ID 从数据库获取完整的文章内容，包括作者信息、分类、标签等关联数据。

#### 场景：获取存在的文章
- **当** 请求已发布的文章详情时
- **那么** 系统应返回文章的完整内容
- **并且** 系统应包含关联的作者信息
- **并且** 系统应包含关联的分类信息
- **并且** 系统应包含关联的标签列表

#### 场景：请求不存在的文章
- **当** 请求不存在的文章 ID 时
- **那么** 系统应返回 null
- **并且** 页面应显示 404

#### 场景：请求未发布的文章
- **当** 普通用户请求未发布的文章时
- **那么** 系统应返回 null
- **并且** 页面应显示 404

### 需求：获取分类列表
系统应提供获取所有可用分类的能力。

#### 场景：获取所有分类
- **当** 前台需要展示分类列表时
- **那么** 系统应返回所有分类
- **并且** 每个分类应包含 id、name、slug、description

### 需求：获取标签列表
系统应提供获取所有可用标签的能力。

#### 场景：获取所有标签
- **当** 前台需要展示标签列表时
- **那么** 系统应返回所有标签
- **并且** 每个标签应包含 id、name、slug

### 需求：获取博客作者信息
系统应提供从数据库或设置中获取博客作者信息的能力。

#### 场景：获取作者信息
- **当** 前台需要展示作者信息时
- **那么** 系统应返回作者信息
- **并且** 作者信息应包含 name、avatar、bio、location、email、social

### 需求：支持静态页面生成（SSG）
系统应支持在构建时预先生成静态页面，提高访问速度。

#### 场景：生成静态文章页面
- **当** 系统构建时
- **那么** 系统应获取所有已发布文章的 ID
- **并且** 系统应为每篇文章生成静态 HTML

#### 场景：增量静态再生成（ISR）
- **当** 文章内容更新时
- **那么** 系统应支持增量再生成对应的静态页面
