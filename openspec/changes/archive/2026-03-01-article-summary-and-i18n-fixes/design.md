## Context

当前系统存在以下问题：

1. **AI 摘要功能不一致**：文章编辑页面（`app/admin/posts/[id]/edit/page.tsx`）有完整的 AI 摘要生成功能，包括状态轮询、生成按钮和重新生成功能。但新建文章页面（`app/admin/posts/new/page.tsx`）完全缺少此功能，用户只能在创建文章后进入编辑页面才能使用摘要生成。

2. **列表页状态显示异常**：文章列表页使用 `AISummaryStatusIcon` 显示 AI 摘要状态，但状态值可能没有正确映射到枚举类型，导致显示不正确。

3. **国际化切换失效**：国际化配置文件可能存在类型不匹配问题，导致语言切换器无法正常工作。

## Goals / Non-Goals

**Goals:**
- 创建可复用的 AI 摘要编辑器组件，在新建和编辑页面共享
- 修复列表页的 AI 摘要状态显示逻辑
- 修复国际化配置和切换功能
- 提供新建页面"自动生成摘要"选项，优化用户体验
- 确保每项任务完成后可独立测试

**Non-Goals:**
- 修改 AI 服务后端逻辑
- 改变现有的摘要生成 API 接口
- 重构整个国际化系统（仅修复当前问题）

## Decisions

### 1. AI 摘要组件提取

**决策**：创建新的 `AISummaryEditor` 组件，封装 AI 摘要的所有功能

**理由**：
- 当前编辑页的 AI 摘要代码约 300 行，包含状态管理、轮询逻辑、UI 显示
- 提取为组件可以在新建和编辑页面复用
- 组件接受 `postId` 作为 prop，在新建页面暂时传入 `null`（保存后获得 ID 再启用）

**组件接口**：
```typescript
interface AISummaryEditorProps {
  postId: string | null  // 新建时为 null
  onSummaryGenerated?: (summary: string) => void
}
```

### 2. 新建页面的工作流

**决策**：新建页面提供"保存后自动生成摘要"勾选框

**理由**：
- AI 摘要生成 API 需要文章 ID，新建页面在保存前没有文章 ID
- 用户可能希望保存后立即生成摘要，而不需要手动操作
- 提供勾选框让用户自主选择，兼顾灵活性和便捷性

**工作流**：
1. 用户填写标题和内容
2. 勾选"保存后自动生成摘要"（可选）
3. 点击"保存草稿"或"发布"
4. 如果勾选了自动生成：保存成功后跳转到编辑页面并自动触发摘要生成
5. 如果未勾选：保存成功后跳转到列表页面（草稿）或保持当前页面（发布）

### 3. 列表页状态修复

**决策**：在 `AISummaryStatusIcon` 组件中添加防御性处理

**理由**：
- 数据库中的 `aiSummaryStatus` 可能是字符串而非枚举
- 组件应能处理无效的状态值，默认显示为 `pending`

**实现**：
```typescript
const current = config[status as SummaryStatus] || config[SummaryStatus.PENDING]
```

### 4. 国际化修复策略

**决策**：检查并修复国际化配置，确保翻译文件正确加载

**理由**：
- 国际化切换失效可能是配置问题，而非类型问题
- 翻译文件是常量对象，TypeScript 可以通过类型推断进行检查
- 不需要显式添加类型注释

**修复步骤**：
1. 检查国际化配置文件（如 `i18n` 配置）是否正确
2. 验证 `locales/en/index.ts` 和 `locales/zh/index.ts` 的导出
3. 检查语言切换器组件的实现
4. 测试语言切换功能是否正常工作

## Risks / Trade-offs

### Risk 1: 新建页面摘要功能体验
**风险**：用户在新建页面无法立即生成摘要，需要先保存

**缓解措施**：
- 在保存成功后自动跳转到编辑页面
- 编辑页面自动加载并可以开始生成摘要

### Risk 2: 组件提取可能引入 Bug
**风险**：将现有代码提取为组件可能破坏现有功能

**缓解措施**：
- 保持编辑页面功能不变，先测试新组件
- 使用渐进式迁移：先在编辑页面验证新组件，再应用到新建页面

### Risk 3: 国际化修复可能影响其他页面
**风险**：修改类型定义可能影响使用翻译的其他组件

**缓解措施**：
- 仅添加缺失的类型定义，不删除现有内容
- 修改后测试多个页面的翻译显示

## Migration Plan

### 阶段 1：修复国际化（优先）
1. 检查国际化配置文件
2. 验证翻译文件导出是否正确
3. 测试语言切换器功能

### 阶段 2：修复列表页状态
1. 修复 `AISummaryStatusIcon` 组件的状态处理
2. 测试文章列表页的状态显示

### 阶段 3：添加新建页面摘要功能
1. 创建 `AISummaryEditor` 组件
2. 在编辑页面集成并测试
3. 在新建页面添加"自动生成摘要"勾选框
4. 实现保存后的条件跳转逻辑（勾选则跳转编辑页并生成，否则跳转列表页）

### 回滚策略
- 每个阶段完成后提交代码
- 如果某阶段出现问题，可回滚到前一阶段
- 不修改后端 API，便于回滚

## Open Questions

1. **AI 摘要组件是否需要支持手动编辑？**
   - 编辑页面已支持手动编辑摘要内容
   - 新组件应保留此功能
   - **决定**：保留手动编辑功能

2. **国际化切换失效的具体原因？**
   - 需要在实施阶段排查
   - 可能原因：配置文件路径错误、中间件问题、翻译文件导出问题
   - **决定**：按顺序排查并修复
