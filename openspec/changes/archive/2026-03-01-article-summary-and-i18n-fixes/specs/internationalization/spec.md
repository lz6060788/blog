## MODIFIED Requirements

### Requirement: 翻译文件结构
系统应在 locales 目录中组织翻译文件，每个语言环境对应一个文件。

#### 场景：翻译文件组织
- **当** 创建翻译文件时
- **那么** 系统应为英语翻译创建 locales/en/index.ts
- **并且** 系统应为中文翻译创建 locales/zh/index.ts
- **并且** 每个文件应导出翻译对象（作为常量）
- **并且** 类型定义文件应位于 locales/types.ts 用于类型推断

#### 场景：翻译文件加载
- **当** 系统加载翻译文件时
- **那么** 系统应正确导入 locales/en/index.ts 和 locales/zh/index.ts
- **并且** TypeScript 应通过类型推断验证翻译对象的类型

### Requirement: 语言切换器组件
系统应提供语言切换器组件，允许用户在支持的语言环境之间切换。

#### 场景：语言切换器显示
- **当** 渲染语言切换器时
- **那么** 系统应显示可用的语言选项（英语、中文）
- **并且** 系统应指示当前活动的语言
- **并且** 点击语言选项应切换到对应语言

#### 场景：语言切换功能
- **当** 用户点击语言切换器中的语言选项
- **那么** 系统应更新当前语言环境
- **并且** 系统应重新加载页面内容
- **并且** 页面上的所有文本应更新为新语言

#### 场景：语言偏好持久化
- **当** 用户更改语言时
- **那么** 系统应将偏好保存到 localStorage
- **并且** 系统应在后续访问中使用保存的偏好
- **并且** 保存的键名应为 'preferred-locale'

### Requirement: 翻译键一致性
系统应确保所有翻译文件包含相同的键结构。

#### 场景：新增翻译键
- **当** 在一个语言文件中添加新的翻译键
- **那么** 系统应在所有其他语言文件中添加对应的键
- **并且** 系统应在 types.ts 中更新 Messages 接口

#### 场景：翻译键缺失处理
- **当** 请求的翻译键在当前语言中不存在
- **那么** 系统应回退到默认语言（英语）
- **并且** 系统应在控制台记录警告

## ADDED Requirements

### Requirement: 翻译类型支持
系统应提供 TypeScript 类型定义用于类型推断和导入时的类型检查。

#### 场景：类型定义文件
- **当** 系统需要翻译类型时
- **那么** 系统应从 locales/types.ts 导出 Messages 类型
- **并且** 系统应从 locales/types.ts 导出 Locale 类型
- **并且** 翻译文件作为常量导出，不需要显式类型注释
