# AI Cover Generation

## ADDED Requirements

### Requirement: AI 封面生成服务
系统应提供文章封面图片自动生成服务，根据文章内容生成高质量封面。

#### 场景：调用封面生成服务
- **当** 用户点击"生成 AI 封面"按钮时
- **则** 系统调用 AI 封面生成服务
- **且** 传入文章标题、内容、标签作为输入
- **且** 返回生成的封面图片 URL

#### 场景：封面生成 Prompt 模板
- **当** AI 服务生成封面时
- **则** 使用预定义的 Prompt 模板
- **且** Prompt 要求 AI 生成与文章主题相关的视觉元素
- **且** Prompt 指定图片尺寸（建议 16:9 比例，1200x675 或更大）
- **且** Prompt 指定风格（简约、现代、专业）

#### 场景：封面风格一致性
- **当** 多篇文章生成封面时
- **则** AI 应保持视觉风格的一致性
- **且** 使用统一的色系和设计语言
- **且** 支持用户配置封面风格偏好（可选）

### Requirement: 封面生成状态管理
系统应管理封面生成状态，包括生成中、已完成、失败等状态。

#### 场景：按钮触发生成
- **当** 用户在文章编辑页点击"生成 AI 封面"按钮时
- **则** 系统调用 `POST /api/admin/posts/[id]/generate-cover` API
- **且** 设置 `ai_cover_status` 为 'generating'
- **且** 立即返回响应，不等待 AI 生成完成

#### 场景：生成期间状态展示
- **当** `ai_cover_status` 为 'generating' 时
- **则** 显示"封面生成中..."加载状态
- **且** 隐藏"生成 AI 封面"按钮
- **且** 禁用封面相关编辑功能

#### 场景：生成完成解锁
- **当** AI 封面生成完成（成功或失败）时
- **则** 设置 `ai_cover_status` 为 'done' 或 'failed'
- **且** 更新 `cover_image_url` 和 `ai_cover_generated_at` 字段（成功时）
- **且** 更新 `ai_cover_prompt` 字段记录使用的 Prompt
- **且** 显示生成的封面或错误提示
- **且** 显示"重新生成"按钮

#### 场景：允许重新生成
- **当** 用户再次点击"生成 AI 封面"按钮（或"重新生成"按钮）时
- **则** 系统重新触发生成流程
- **且** 覆盖原有的封面图片

#### 场景：前端轮询状态
- **当** 前端触发封面生成后
- **则** 每 3 秒轮询一次 `GET /api/admin/posts/[id]/ai-cover-status`
- **当** 状态变为 'done' 或 'failed' 时
- **则** 停止轮询并更新 UI

### Requirement: 封面图片存储
系统应将 AI 生成的封面图片存储到对象存储服务。

#### 场景：生成后上传到对象存储
- **当** AI 返回生成的图片（URL 或 Base64）时
- **则** 系统将图片上传到腾讯云 COS
- **则** 使用规范的文件路径：`covers/{year}/{month}/{post_id}_{timestamp}.png`
- **且** 将 COS URL 存储到 `cover_image_url` 字段

#### 场景：图片格式和大小限制
- **当** AI 返回图片时
- **则** 系统验证图片格式（支持 PNG、JPG、WEBP）
- **且** 验证图片大小（不超过 5MB）
- **且** 如需要，进行图片压缩和格式转换

### Requirement: 手动上传封面
系统应支持用户手动上传封面图片，替换 AI 生成的封面。

#### 场景：手动上传封面
- **当** 用户点击"上传封面"按钮时
- **则** 系统打开文件选择器
- **且** 支持拖拽上传
- **且** 上传完成后更新 `cover_image_url`
- **且** 设置 `ai_cover_status` 为 'manual'

#### 场景：手动上传后清除 AI 标记
- **当** 用户手动上传封面时
- **则** 清除 `ai_cover_generated_at` 和 `ai_cover_prompt` 字段
- **且** 标记来源为手动上传

### Requirement: 封面预览和编辑
系统应在文章编辑页面提供封面预览和编辑功能。

#### 场景：展示封面预览
- **当** 文章有封面图片时
- **则** 在编辑页面顶部或侧边栏展示封面预览
- **且** 显示封面来源标签（AI 生成/手动上传）
- **且** 提供编辑操作（重新生成/删除/更换）

#### 场景：删除封面
- **当** 用户点击"删除封面"按钮时
- **则** 系统确认删除操作
- **且** 清空 `cover_image_url` 字段
- **且** 重置 `ai_cover_status` 为 'pending'

### Requirement: 错误处理与重试
系统应实现完善的错误处理和重试机制。

#### 场景：AI 调用失败重试
- **当** AI 服务调用失败时（网络错误、超时、5xx 错误）
- **则** 系统自动重试最多 2 次
- **且** 重试采用指数退避策略（1 秒、2 秒）
- **且** 记录每次重试到日志

#### 场景：重试失败后提示
- **当** 重试 2 次后仍然失败时
- **则** 设置 `ai_cover_status` 为 'failed'
- **且** 记录错误信息到 `ai_call_logs` 表
- **且** 显示错误提示"封面生成失败：{错误原因}"
- **且** 允许用户重试

#### 场景：图片上传失败处理
- **当** 图片上传到 COS 失败时
- **则** 保留 AI 生成的图片（如可用）
- **且** 记录错误到日志
- **且** 提示用户稍后重试或手动上传

#### 场景：配置未检查提示
- **当** AI 服务未配置图像生成模型时
- **则** 返回错误提示"请先在管理后台配置 AI 图像生成模型"
- **且** 提供跳转链接到配置页面

### Requirement: 生成频率限制
系统应限制封面生成频率，控制 API 成本。

#### 场景：每篇文章生成次数限制
- **当** 用户请求生成封面时
- **则** 系统检查过去 24 小时内的生成次数
- **且** 单篇文章每天最多生成 5 次
- **且** 超过限制时提示"今日生成次数已达上限，请明天再试"

#### 场景：全局生成频率限制
- **当** 系统处理封面生成请求时
- **则** 系统检查全局生成速率
- **且** 全局每分钟最多生成 10 张封面
- **且** 超过限制时返回 429 错误，提示"服务器繁忙，请稍后再试"

### Requirement: 封面在文章列表展示
系统应在文章列表和文章详情页展示封面图片。

#### 场景：文章列表展示封面
- **当** 用户浏览文章列表时
- **则** 文章卡片应展示封面图片（如有）
- **且** 使用懒加载优化性能
- **且** 支持响应式尺寸

#### 场景：文章详情页展示封面
- **当** 用户查看文章详情时
- **则** 文章顶部应展示封面图片（如有）
- **且** 作为文章的主视觉元素

#### 场景：社交媒体分享卡片
- **当** 文章被分享到社交媒体时
- **则** 使用封面图片作为 Open Graph Image
- **且** 提升分享效果和点击率
