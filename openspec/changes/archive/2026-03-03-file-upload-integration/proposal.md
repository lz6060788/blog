## 背景原因

当前博客系统缺少文件上传功能，用户无法在文章中插入图片或附件。随着内容丰富度需求的增加，集成对象存储服务以支持图片和文档上传已成为必要功能。腾讯云 COS 提供稳定、高可用的对象存储服务，适合作为博客系统的文件存储后端。

## 变更内容

- **新增后端文件上传模块**：集成腾讯云 COS SDK，实现文件上传、删除、权限管理等核心功能
- **新增文件上传 API**：提供统一的文件上传接口，支持图片、文档等多种文件类型
- **新增前端上传 Hook**：创建通用的 `useFileUpload` hook，封装上传逻辑和状态管理
- **集成 Cherry Markdown 编辑器**：配置编辑器的图片上传功能，对接对象存储服务
- **新增文件类型验证**：实现文件大小、类型的安全校验机制
- **新增上传进度反馈**：提供用户友好的上传进度展示

## Capabilities

### 新增能力
- `file-upload-service`: 后端文件上传服务，封装腾讯云 COS 集成，提供文件上传、删除、URL 生成等核心功能
- `file-upload-api`: 前端文件上传 API 接口定义，包括请求类型、响应格式和错误处理
- `file-upload-hook`: 前端通用上传 Hook，提供上传状态管理、进度追踪和错误处理
- `markdown-image-upload`: Cherry Markdown 编辑器图片上传集成，支持拖拽上传和粘贴上传

### 修改的能力
- 无（此为新功能，不修改现有能力的行为要求）

## 影响范围

**后端影响**：
- 新增腾讯云 COS SDK 依赖（`cos-nodejs-sdk-v5`）
- 新增文件上传路由和控制器模块
- 需要配置腾讯云 COS 认证信息（SecretId、SecretKey、Bucket 等）
- 新增文件中间件处理（类型验证、大小限制）

**前端影响**：
- 新增 `@/lib/upload` 模块，包含上传 Hook 和 API 定义
- 新增 `@/components/uploader` 组件（可选，用于独立上传场景）
- 修改 Cherry Markdown 编辑器配置，集成自定义上传函数
- 新增上传状态类型定义和错误处理

**外部依赖**：
- 腾讯云 COS 服务
- 环境变量配置（COS 配置信息）

**安全考虑**：
- 文件类型白名单验证
- 文件大小限制
- 上传频率限制（可选）
- 私有 Bucket 访问权限控制
