## 新增需求

### 需求：上传 API 端点
系统必须提供用于文件上传的 REST API 端点。

#### 场景：POST /api/upload
- **当** 客户端向 `/api/upload` 发送包含 multipart/form-data 的 POST 请求
- **并且** 请求包含带有有效文件数据的文件字段
- **并且** 请求包含有效的认证令牌
- **则** 系统必须返回 HTTP 201 Created
- **并且** 响应体必须包含：
  - `url`：上传文件的公开 URL
  - `key`：COS 对象键
  - `filename`：原始文件名
  - `size`：文件大小（字节）
  - `type`：文件的 MIME 类型

#### 场景：无认证上传
- **当** 客户端发送没有认证令牌的 POST 请求
- **则** 系统必须返回 HTTP 401 Unauthorized
- **并且** 响应必须包含错误消息

#### 场景：上传进度追踪
- **当** 客户端发送带有 `X-Upload-ID` 头的 POST 请求
- **则** 系统必须支持通过上传 ID 进行进度追踪
- **并且** 可选地在响应头中返回上传进度

### 需求：删除 API 端点
系统必须提供用于文件删除的 REST API 端点。

#### 场景：DELETE /api/upload/:key
- **当** 认证用户向 `/api/upload/:key` 发送 DELETE 请求
- **并且** 用户拥有该文件或具有管理员权限
- **则** 系统必须返回 HTTP 200 OK
- **并且** 响应必须包含成功消息
- **并且** 文件必须从 COS 存储中移除

#### 场景：删除不存在的文件
- **当** 用户发送针对不存在文件键的 DELETE 请求
- **则** 系统必须返回 HTTP 404 Not Found
- **并且** 响应必须包含错误消息

#### 场景：未授权删除
- **当** 用户尝试删除他们不拥有的文件
- **则** 系统必须返回 HTTP 403 Forbidden
- **并且** 响应必须包含错误消息

### 需求：错误响应格式
系统必须为所有 API 端点返回一致的错误响应。

#### 场景：验证错误
- **当** 请求未通过验证（如文件过大、类型无效）
- **则** 系统必须返回 HTTP 400 Bad Request
- **并且** 响应必须包含：
  - `error`：错误代码（如 "FILE_TOO_LARGE"、"INVALID_FILE_TYPE"）
  - `message`：人类可读的错误消息
  - `details`：其他验证详细信息

#### 场景：服务器错误
- **当** 上传期间发生意外服务器错误
- **则** 系统必须返回 HTTP 500 Internal Server Error
- **并且** 响应必须包含：
  - `error`：错误代码（如 "UPLOAD_FAILED"）
  - `message`：通用错误消息（不暴露内部细节）

### 需求：请求速率限制
系统必须在上传端点上实施速率限制。

#### 场景：超过速率限制
- **当** 用户超过上传速率限制（如每分钟 10 次上传）
- **则** 系统必须返回 HTTP 429 Too Many Requests
- **并且** 响应必须包含：
  - `retryAfter`：用户可以重试前的秒数

#### 场景：速率限制范围内
- **当** 用户在速率限制内进行上传
- **则** 系统必须正常处理所有请求
- **并且** 不返回速率限制错误
