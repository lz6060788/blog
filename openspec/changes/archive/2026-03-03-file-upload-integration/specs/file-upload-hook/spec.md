## 新增需求

### 需求：useFileUpload Hook
系统必须提供 React Hook，用于管理文件上传和状态追踪。

#### 场景：初始化 Hook
- **当** 组件调用 `useFileUpload()`
- **则** Hook 必须返回包含以下内容的对象：
  - `upload`：启动文件上传的函数
  - `isUploading`：指示上传进行中的布尔值
  - `progress`：数字（0-100），指示上传进度
  - `error`：字符串或 null，表示错误消息
  - `reset`：重置上传状态的函数

#### 场景：成功上传
- **当** 用户使用有效文件调用 `upload(file)`
- **则** 上传期间 `isUploading` 必须为 true
- **并且** `progress` 必须随着上传进度更新
- **并且** 完成后，`isUploading` 必须为 false
- **并且** `error` 必须为 null
- **并且** Hook 必须返回上传结果（URL、元数据）

#### 场景：上传出错
- **当** 用户调用 `upload(file)` 并且上传失败
- **则** `isUploading` 必须为 false
- **并且** `error` 必须包含错误消息
- **并且** `progress` 必须重置为 0

### 需求：进度追踪
Hook 必须提供实时上传进度更新。

#### 场景：上传期间进度更新
- **当** 文件上传正在进行
- **则** `progress` 值必须定期更新
- **并且** 进度必须是 0 到 100 之间的数字
- **并且** 进度必须反映实际上传百分比

#### 场景：完成时的进度
- **当** 上传成功完成
- **则** `progress` 必须为 100
- **并且** 保持为 100，直到重置或新上传开始

### 需求：文件验证
Hook 必须在启动上传之前验证文件。

#### 场景：文件大小验证
- **当** 用户尝试上传超过大小限制的文件
- **则** Hook 必须立即拒绝上传
- **并且** 设置 `error` 指示文件大小超出
- **并且** 不启动 API 请求

#### 场景：文件类型验证
- **当** 用户尝试上传类型无效的文件
- **则** Hook 必须立即拒绝上传
- **并且** 设置 `error` 指示不支持的文件类型
- **并且** 不启动 API 请求

### 需求：状态重置
Hook 必须提供重置上传状态的功能。

#### 场景：成功上传后重置
- **当** 用户在成功上传后调用 `reset()`
- **则** `isUploading` 必须为 false
- **并且** `progress` 必须为 0
- **并且** `error` 必须为 null

#### 场景：失败上传后重置
- **当** 用户在失败上传后调用 `reset()`
- **则** `isUploading` 必须为 false
- **并且** `progress` 必须为 0
- **并且** `error` 必须为 null

### 需求：多文件支持
Hook 必须支持并发上传多个文件。

#### 场景：上传多个文件
- **当** 用户使用多个文件调用 `upload([file1, file2, file3])`
- **则** Hook 必须上传所有文件
- **并且** 分别追踪每个文件的进度
- **并且** 返回上传结果数组

#### 场景：部分上传失败
- **当** 上传多个文件并且部分失败
- **则** Hook 必须完成成功的上传
- **并且** 报告失败上传的错误
- **并且** 提供部分结果，包含每个文件的成功/失败状态

### 需求：取消支持
Hook 必须支持取消正在进行的上传。

#### 场景：取消上传
- **当** 用户在上传期间调用 `cancel()`
- **则** Hook 必须中止上传请求
- **并且** 设置 `isUploading` 为 false
- **并且** 重置 `progress` 为 0
- **并且** 设置 `error` 指示取消

#### 场景：取消时清理
- **当** 上传被取消
- **则** Hook 必须清理所有临时资源
- **并且** 不在服务器上留下部分文件
