## 新增需求

### 需求：上传文件到 COS
系统必须提供后端服务，用于处理文件上传到腾讯云 COS。

#### 场景：成功上传图片
- **当** 用户上传有效的图片文件（JPG、PNG、GIF、WEBP），大小不超过 5MB
- **则** 系统必须将文件上传到 COS
- **并且** 返回可通过 HTTPS 访问的公开 URL
- **并且** 存储文件元数据（文件名、大小、类型、上传时间）

#### 场景：文件大小验证
- **当** 用户尝试上传超过 5MB 的文件
- **则** 系统必须拒绝上传
- **并且** 返回错误消息，指示文件大小限制

#### 场景：无效文件类型
- **当** 用户尝试上传未经授权的扩展名文件（如 .exe、.sh）
- **则** 系统必须拒绝上传
- **并且** 返回错误消息，指示不支持的文件类型

### 需求：从 COS 删除文件
系统必须提供从腾讯云 COS 删除文件的功能。

#### 场景：成功删除文件
- **当** 授权用户通过对象键请求删除文件
- **则** 系统必须从 COS 删除文件
- **并且** 返回成功确认

#### 场景：删除不存在的文件
- **当** 用户请求删除不存在的文件
- **则** 系统必须返回错误，指示未找到文件
- **并且** 不影响其他文件

### 需求：生成签名 URL
系统必须为私有文件的临时访问生成签名 URL。

#### 场景：生成有效的签名 URL
- **当** 用户为现有文件请求签名 URL
- **则** 系统必须生成带有临时访问令牌的 URL
- **并且** URL 必须在配置的时间段后过期（默认 1 小时）

#### 场景：无效的文件键
- **当** 用户为不存在的文件请求签名 URL
- **则** 系统必须返回错误，指示未找到文件

### 需求：文件元数据存储
系统必须在数据库中维护上传文件的元数据。

#### 场景：存储文件元数据
- **当** 文件成功上传
- **则** 系统必须存储以下元数据：
  - 原始文件名
  - COS 对象键
  - 文件大小（字节）
  - MIME 类型
  - 上传时间戳
  - 上传者用户 ID

#### 场景：检索文件元数据
- **当** 用户通过对象键查询文件元数据
- **则** 系统必须返回该文件的所有存储元数据

### 需求：安全和认证
系统必须对文件操作执行安全措施。

#### 场景：需要认证
- **当** 未认证的用户尝试上传文件
- **则** 系统必须拒绝请求
- **并且** 返回 401 Unauthorized 错误

#### 场景：授权检查
- **当** 用户尝试删除他们未上传的文件
- **则** 系统必须拒绝请求
- **并且** 返回 403 Forbidden 错误
