## 新增需求

### 需求：Cherry Markdown 图片上传集成
系统必须将 Cherry Markdown 编辑器的图片上传功能与文件上传服务集成。

#### 场景：拖放图片上传
- **当** 用户将图片文件拖放到 Cherry Markdown 编辑器中
- **则** 编辑器必须使用配置的上传函数自动上传图片
- **并且** 在光标位置插入图片 markdown 语法 `![alt text](url)`
- **并且** 用来自 COS 的实际图片 URL 替换占位符

#### 场景：粘贴图片上传
- **当** 用户将图片从剪贴板粘贴到 Cherry Markdown 编辑器中
- **则** 编辑器必须使用配置的上传函数自动上传图片
- **并且** 插入带有 COS URL 的图片 markdown 语法

#### 场景：工具栏图片按钮上传
- **当** 用户点击编辑器工具栏中的图片按钮
- **并且** 从文件选择器中选择图片文件
- **则** 编辑器必须使用配置的上传函数上传图片
- **并且** 插入带有 COS URL 的图片 markdown 语法

### 需求：上传配置
Cherry Markdown 编辑器必须配置自定义上传处理程序。

#### 场景：自定义上传处理程序
- **当** 初始化 Cherry Markdown 编辑器
- **则** 编辑器必须配置自定义 `upload` 函数
- **并且** 函数必须使用 `useFileUpload` Hook 或直接 API 调用
- **并且** 以 Cherry Markdown 期望的格式返回图片 URL

#### 场景：上传错误处理
- **当** 编辑器上传期间图片上传失败
- **则** 编辑器必须向用户显示错误消息
- **并且** 不插入损坏的图片 markdown 语法
- **并且** 允许用户重试上传

### 需求：上传进度指示
编辑器必须在上传期间提供视觉反馈。

#### 场景：显示上传进度
- **当** 正在上传图片
- **则** 编辑器必须显示加载指示器
- **并且** 可选地显示上传进度百分比
- **并且** 禁用额外上传，直到当前上传完成

#### 场景：上传完成
- **当** 图片上传成功完成
- **则** 加载指示器必须被移除
- **并且** 图片必须在编辑器预览中渲染
- **并且** 编辑器必须准备好进行额外上传

### 需求：图片预览
系统必须在编辑器预览中渲染上传的图片。

#### 场景：预览上传的图片
- **当** 图片成功上传并插入
- **则** 编辑器预览必须使用 COS URL 渲染图片
- **并且** 图片必须无认证问题地加载
- **并且** 图片必须响应于编辑器容器宽度

#### 场景：处理图片加载失败
- **当** 图片在预览中加载失败
- **则** 编辑器必须显示替代文本占位符
- **并且** 显示损坏图片的错误指示器

### 需求：文件类型限制
Cherry Markdown 上传集成必须仅接受图片文件。

#### 场景：仅接受图片文件
- **当** 用户尝试通过编辑器上传非图片文件
- **则** 系统必须拒绝上传
- **并且** 显示错误消息，指示仅允许图片
- **并且** 支持的类型必须包括：JPG、PNG、GIF、WEBP、SVG

#### 场景：文件大小验证
- **当** 用户尝试通过编辑器上传超过 5MB 的图片
- **则** 系统必须拒绝上传
- **并且** 显示错误消息，指示文件大小限制

### 需求：多图片处理
编辑器必须支持顺序上传多个图片。

#### 场景：顺序图片上传
- **当** 用户一个接一个地上传多个图片
- **则** 每个图片必须独立上传
- **并且** 每个图片必须插入到其各自的光标位置
- **并且** 上传必须不阻止编辑器进行其他操作

#### 场景：批量图片粘贴
- **当** 用户一次粘贴多个图片
- **则** 编辑器必须上传所有图片
- **并且** 顺序插入每个图片 markdown
- **并且** 保持粘贴图片的顺序
