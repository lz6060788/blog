# 主题系统规范变更

## REMOVED Requirements

### 需求：主题配置文件
**原因**: 该需求要求创建 `theme.config.ts` 文件，但实际实现中发现该文件未被使用，与 `app/globals.css` 的 CSS 变量配置重复。为简化主题系统，统一使用 CSS 变量作为唯一配置源。

**迁移**: 无需迁移，该文件未被任何代码使用。直接删除即可。

原需求内容：
```
系统应提供一个集中化的主题配置文件，使用 CSS 自定义属性定义所有颜色变量。

#### 场景：主题配置创建
- **当** 实现主题系统时
- **那么** 系统应在项目根目录创建 theme.config.ts 文件
- **并且** 配置应导出浅色和深色模式的颜色定义
- **并且** 系统应从此配置生成 CSS 自定义属性
```

## MODIFIED Requirements

### 需求：CSS 自定义属性

原需求描述：系统应使用 CSS 自定义属性（CSS 变量）来设置所有主题颜色。

#### 场景：主题配置的唯一来源

- **当** 定义主题颜色时
- **那么** 系统应仅在 `app/globals.css` 中定义 CSS 变量
- **并且** 系统不创建额外的 TypeScript 配置文件（如 theme.config.ts）
- **并且** Tailwind 配置直接引用 CSS 变量（如 `var(--bg-canvas)`）
- **并且** CSS 变量是主题配置的唯一事实来源

#### 场景：CSS 变量的运行时特性

- **当** 用户切换主题（深色/浅色模式）
- **那么** 系统通过切换 `.dark` 类来更新 CSS 变量值
- **并且** 所有引用这些变量的组件自动更新颜色
- **并且** 无需重新构建或重新加载页面
