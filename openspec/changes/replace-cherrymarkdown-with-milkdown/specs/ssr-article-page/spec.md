## MODIFIED Requirements

### Requirement: 客户端组件协调
系统应正确协调服务端组件和客户端组件，确保主题切换等客户端功能正常工作。文章内容预览组件从 CherryPreview 变更为 MilkdownPreview，该组件支持服务端渲染。

#### Scenario: 主题状态在服务端和客户端同步
- **当** 页面初始渲染时
- **那么** 服务端应使用默认主题渲染初始 HTML
- **并且** 客户端水合后应恢复用户的主题偏好
- **并且** MilkdownPreview 组件应能响应客户端主题变化

#### Scenario: MilkdownPreview 嵌入到文章详情页
- **当** 文章详情页渲染时
- **那么** MilkdownPreview 应作为可支持 SSR 的组件嵌入
- **并且** MilkdownPreview 应接收文章内容和当前主题作为 props
- **并且** MilkdownPreview 应使用只读模式渲染完整内容
- **并且** MilkdownPreview 应在服务端渲染时输出完整的 HTML
- **并且** 搜索引擎爬虫应能从 HTML 中直接读取文章内容

#### Scenario: 服务端渲染文章内容
- **当** 用户请求文章详情页时
- **那么** 系统应在服务器上获取文章数据
- **并且** 系统应在服务器上使用 MilkdownPreview 渲染文章内容的初始 HTML
- **并且** 系统应将完整的 HTML 发送到客户端
- **并且** HTML 应包含所有格式化的 Markdown 内容（标题、列表、代码块等）

#### Scenario: 搜索引擎爬虫可获取完整内容
- **当** 搜索引擎爬虫访问文章页面时
- **那么** 爬虫应能获取服务端渲染后的文章完整内容
- **并且** 爬虫应能读取文章的元数据（标题、描述、标签）
- **并且** 文章内容应以 HTML 格式直接存在于页面源码中
- **并且** 无需执行 JavaScript 即可获取文章内容

#### Scenario: 客户端交互组件独立水合
- **当** 页面在客户端加载完成时
- **那么** 系统应仅对需要客户端交互的组件进行水合（hydration）
- **并且** MilkdownPreview 组件应保持服务端渲染的内容，仅在需要时进行水合
- **并且** 主题切换等功能应正常工作
