## 需求：管理端主题提供程序
管理端布局应被 ThemeProvider 包装，使管理端能够访问和切换主题状态。

#### 场景：管理端加载时应用已保存主题
- **当** 用户进入管理端页面
- **且** localStorage 中存在已保存的主题配置
- **则** 管理端应应用已保存的主题
- **且** 主题与主站保持一致

#### 场景：管理端默认为浅色主题
- **当** 用户首次进入管理端
- **且** localStorage 中不存在主题配置
- **则** 管理端应应用浅色主题

## 需求：管理端主题切换组件
管理端顶部栏应提供主题切换组件，允许用户在浅色、深色和自动模式之间切换。

#### 场景：显示主题切换按钮
- **当** 用户查看管理端顶部栏
- **则** 应在用户头像左侧显示主题切换组件
- **且** 组件包含浅色、深色、自动三个选项
- **且** 当前选中的主题应有视觉指示

#### 场景：切换主题
- **当** 用户点击主题切换组件中的任一选项
- **则** 管理端应立即应用新主题
- **且** 主题配置应保存到 localStorage
- **且** 保存键名应为 "blog-theme"

#### 场景：主题状态同步到主站
- **当** 用户在管理端更改主题
- **则** 主站应能读取到新的主题配置
- **且** 返回主站时应显示新选择的主题

#### 场景：主站主题状态同步到管理端
- **当** 用户在主站更改主题后进入管理端
- **则** 管理端应显示主站最后选择的主题

## 需求：主题切换组件移动端适配
主题切换组件在移动端应保持可用性和可访问性。

#### 场景：移动端显示主题切换
- **当** 用户在移动设备上查看管理端
- **则** 主题切换组件应正常显示
- **且** 组件大小应适合触摸操作
- **且** 不应与其他控件重叠

## 需求：ThemeProvider 配置一致性
管理端和主站的 ThemeProvider 应使用相同配置，确保主题状态正确同步。

#### 场景：ThemeProvider 配置
- **当** ThemeProvider 在管理端初始化
- **则** 应使用 `attribute="class"` 配置
- **且** 应使用 `defaultTheme="light"` 配置
- **且** 应使用 `enableSystem` 配置（支持自动模式）
- **且** 应使用 `disableTransitionOnChange={false}` 配置（启用主题切换过渡）
