## 新增需求

### 需求：管理员邮箱白名单
系统应仅允许预先配置的管理员邮箱地址登录，拒绝所有其他用户的访问请求。

#### 场景：白名单用户成功登录
- **当** 用户通过 OAuth 登录且邮箱匹配 `ADMIN_EMAIL` 环境变量
- **则** 系统允许用户登录
- **且** 创建有效会话

#### 场景：非白名单用户被拒绝
- **当** 用户通过 OAuth 登录但邮箱不匹配 `ADMIN_EMAIL`
- **则** 系统拒绝登录请求
- **且** 返回错误信息"未授权访问"
- **且** 不创建会话

#### 场景：环境变量未配置
- **当** `ADMIN_EMAIL` 环境变量未设置
- **则** 系统启动时显示错误
- **且** 拒绝所有登录尝试

### 需求：邮箱大小写不敏感验证
系统应以大小写不敏感的方式比较用户邮箱与白名单配置。

#### 场景：邮箱大小写不同但匹配
- **当** `ADMIN_EMAIL` 配置为 `admin@example.com`
- **且** 用户 OAuth 返回邮箱为 `Admin@Example.com`
- **则** 系统将用户邮箱转换为小写后比较
- **且** 允许用户登录

### 需求：OAuth 提供商无关验证
白名单验证应独立于 OAuth 提供商，任何提供商返回的匹配邮箱都应被接受。

#### 场景：使用不同提供商但相同邮箱
- **当** `ADMIN_EMAIL` 为 `admin@example.com`
- **且** 用户首次使用 GitHub 登录（邮箱: admin@example.com）
- **则** 系统允许登录
- **当** 用户后续使用 Google 登录（邮箱: admin@example.com）
- **则** 系统继续允许登录
- **且** 使用同一用户账号
